📚 Detection Algorithm Theory

Before diving into examples, understand the two core detection algorithms.

Algorithm 1

Single-Instance: Wait-For Graph

Single-Instance

For systems where each resource type has only ONE instance.

Method:

  1. Reduce RAG by removing resource nodes
  2. If Pi→Rk and Rk→Pj exists in RAG, add Pi→Pj in WFG
  3. A cycle in WFG = DEADLOCK
  4. Complexity: O(n²)
Example WFG:
P1→P2→P3→P1 (CYCLE → DEADLOCK!)
P4→P2 (no cycle → P4 not deadlocked)
Algorithm 2

Multi-Instance: Detection Algorithm

Multi-Instance

For systems where resource types may have multiple instances.

Data Structures:

  • Available[m] - free instances
  • Allocation[n][m] - current allocation
  • Request[n][m] - outstanding requests

Steps:

  1. Work = Available; Finish[i] = (Allocation[i]==[0])
  2. Find i: Finish[i]=false AND Request[i]≤Work
  3. Work += Allocation[i]; Finish[i]=true; goto 2
  4. Finish[i]=false → Process i is DEADLOCKED
Complexity:
O(m × n²)
